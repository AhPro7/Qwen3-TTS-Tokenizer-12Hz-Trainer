The Goal: Zero-Shot Voice Conversion

Separate WHO is speaking from WHAT is being said — then swap.

🗣️ Speaker (WHO)

Voice timbre, pitch range, breathiness, accent — constant across the utterance

→ Global representation (256-dim)

📝 Content (WHAT)

Phonemes, words, melody, timing — changes every millisecond

→ Per-frame representation (1024-dim)

Without separation: changing the voice also changes the words
With separation: swap only the voice, keep the words perfectly

The 16 Codebooks (RVQ)

Qwen encodes audio using Residual Vector Quantization — 16 layers of successive approximation:

CodebooksWhat They CaptureAnalogy
1–4 (coarse)Phoneme identity, pitch, F0📝 "What word"
5–10 (mid)Formants, prosody, energy🎵 "How it's said"
11–16 (fine)Timbre, vocal tract, breathiness🗣️ "Who says it"

Key: At decode time, all 16 are summed into one vector — the separation is implicit and leaky.

// xcodec2/vq/residual_vq.py
quantized_out = 0.
for idx, layer in enumerate(self.layers):  // all 16
    quantized_out = quantized_out + quantized  // SUM

The Problem: Implicit ≠ Controllable

Why can't we just swap codebooks 11-16 between speakers?

❌

Split codebooks manually

Content leaks into ALL layers. Removing any layer destroys audio quality. Speaker info bleeds into codebooks 1-4.

❌

Retrain separate VQs

Requires retraining the entire encoder from scratch — loses all pretrained knowledge.

❌

External speaker encoder

ECAPA/x-vector trained on different data. Embedding doesn't align with decoder's latent space.

✅

Our approach

Feed all 16 together, learn the separation after in the decoder's own latent space. Minimal params, end-to-end.

Our Solution: DisentangledProjection

Injected between the Pre-Transformer and the HiFiGAN Decoder:

❄️ Quantizer Decode — sum all 16 codebooks
↓
❄️ Pre-Conv + Pre-Transformer (4-layer Qwen2)
↓ hidden [B, T, 1024] — everything mixed
🔥 DisentangledProjection
Speaker: 1024→256→pool→broadcast | Content: 1024→1024 per-frame
↓ speaker_contrib + content_emb
🔥 Upsample + HiFiGAN Decoder → Waveform
// trainer.py — DecoderTrainingWrapper.forward()
hidden = self.decoder.quantizer.decode(codes)   // all 16 summed
hidden = self.decoder.pre_transformer(inputs_embeds=hidden).last_hidden_state

speaker_contrib, content_emb, _ = self.disentangle(hidden)
hidden = speaker_contrib + content_emb  // recombine → decoder

Speaker Path: Extract Voice Identity

Sees all frames → compresses each → then pools into ONE global vector. The output is the same for every frame.

Step-by-step with real numbers:

Input: 5 frames × 1024-dim  (each frame = phoneme + voice mixed)

Step A: Compress 1024 → 256 per frame (speaker_encoder)
Step B: Attention — learn weight per frame (which frames best show the voice?)
   Frame 1: 0.15  Frame 2: 0.25  Frame 3: 0.20  Frame 4: 0.10  Frame 5: 0.30

Step C: Pool — weighted average → ONE vector
   speaker_global = 0.15×f1 + 0.25×f2 + ... = [0.24, 0.67, ...] // 256-dim
   5 frames → 1 vector. Words are GONE. Only voice survives.

Step D: Expand 256 → 1024 (speaker_decoder)
Step E: Broadcast — copy SAME vector to ALL frames
   Frame 1: [0.1, -0.05, ...]  ← same!
   Frame 2: [0.1, -0.05, ...]  ← same!   // "Ahmed's voice" everywhere
   Frame 3: [0.1, -0.05, ...]  ← same!
// The key line — time is destroyed:
return (h * attn).sum(dim=1)  // [B, T, 256] → [B, 256]  TIME GONE

Content Path: Extract What Is Said

Per-frame MLP — each frame processed independently. All temporal detail preserved.

How it works

Frame 1 → MLP → content("م")  ← unique
Frame 2 → MLP → content("ر")  ← unique
Frame 3 → MLP → content("ح")  ← unique
Frame 4 → MLP → content("ب")  ← unique
Frame 5 → MLP → content("ا")  ← unique
// Each frame gets DIFFERENT output!

Voice identity = low-dimensional (~20 attributes: pitch, breathiness, nasality...)

Voice Conversion: The Swap

Normal reconstruction:

Frame 1:  content("م") + speaker(Ahmed) = "م in Ahmed's voice"
Frame 2:  content("ر") + speaker(Ahmed) = "ر in Ahmed's voice"
Frame 3:  content("ح") + speaker(Ahmed) = "ح in Ahmed's voice"

Voice conversion — swap the speaker:

Frame 1:  content("م") + speaker(Sara)  = "م in Sara's voice"  ✅
Frame 2:  content("ر") + speaker(Sara)  = "ر in Sara's voice"  ✅
Frame 3:  content("ح") + speaker(Sara)  = "ح in Sara's voice"  ✅
Same words, different voice!
// voice_convert.py — the swap is literally one line:
content_A = self.disentangle.encode_content(src_hidden)   // A's words
speaker_B = self.disentangle.encode_speaker(tgt_hidden)   // B's voice
combined = speaker_B_contrib + content_A  // B's voice + A's words

Training Loop: GAN Training

Each step: Discriminator judges → Generator improves

🔥 Generator (our model)

Qwen decoder + DisentangledProjection

Goal: produce audio so real the discriminator can't tell

🔍 Discriminator (judge)

MPD + MSD — two separate CNNs

Goal: detect which audio is generated vs real

⚠️ Discarded after training — not used at inference

What's Trained vs Frozen

Quantizer, Pre-Conv, Pre-Transformer❄️ Frozen
DisentangledProjection (2.6M params)🔥 Trained
HiFiGAN Upsample + Decoder🔥 Trained
MPD + MSD Discriminators🔥 Trained then discarded

The 6 Generator Losses

LossWeightWhat It Does
Mel spectrogram15.0"Does it SOUND right?" — compares at 7 frequency scales
Feature matching3.0"Match discriminator's internal features" — stabilizes GAN
Consistency1.0"Don't distort the hidden state" — prevents artifacts
Global RMS1.0"Match overall loudness (dB)"
Adversarial0.3"Fool the discriminator" — adds naturalness
Orthogonality0.05"Speaker ⊥ content" — forces separation

🔑 Our additions: Orthogonality + Consistency

Orthogonality: cos_sim(speaker, content) → 0 = perpendicular = encoding DIFFERENT info. If they overlap, swapping speaker accidentally changes words.

Consistency: MSE(speaker+content, original_hidden) → prevents the decoder from receiving distorted signals early in training.

The Discriminator: Two Judges

🎵 MPD — Multi-Period Discriminator

"Does the audio have natural periodic patterns?"

Examines at periods 2, 3, 5, 7, 11 (all prime — can't alias each other)

Catches: metallic buzzing, pitch artifacts, clicks

📊 MSD — Spectral Discriminator

"Does the audio have natural frequency content?"

Applies STFT at 8 resolutions (FFT 78 → 2296)

Catches: frequency smearing, muffled sound, robot voice

The GAN Game

Discriminator trains:  real audio → score 1.0,  fake → score 0.0
Generator trains:      fake audio → push score toward 1.0 (fool judge)

Healthy:  dr stays ~1.0,  dg slowly climbs 0 → 0.7+
Bad:      dg → 1.0 too fast (disc gave up) or dg stuck at 0 (gen collapsed)

Summary

🎙️ Qwen3-TTS encodes audio into 16 codebooks (RVQ) — all summed together

🧠 We add DisentangledProjection (2.6M params, <2% overhead) to LEARN the separation

🔒 Speaker path: 1024→256→pool→global vector — CAN'T encode content (time destroyed)

📝 Content path: 1024→1024 per-frame MLP — preserves all temporal detail

⊥ Orthogonality loss ensures they encode DIFFERENT information

🎯 Result: swap speaker vector → same words, new voice

Zero-shot voice conversion by design, not by luck.
